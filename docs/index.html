CI-native maintainer intelligence

Maintainer Signal

A zero-dependency GitHub Action and CLI that turns issues and pull requests into project health reports, triage queues, release-note candidates, and optional OpenAI maintainer briefs.

Star on GitHub Quick Start

Built For Maintainers

Issue Triage

Highlights stale threads, first-response gaps, likely labels, and priority candidates.

Release Notes

Collects merged pull requests and turns them into a clean release-note queue.

Read-Only By Default

Designed to run in CI without write access or an external dashboard.

Optional OpenAI

Adds maintainer briefs when OPENAI_API_KEY is configured; deterministic reports still work without it.

Start here: open the sample report, follow the adoption checklist, skim CONTRIBUTING.md, and copy examples/quality-gate.yml if you want a health gate that still uploads a report.

Install

Use Luke2program/maintainer-signal@v0.2.1 in a scheduled GitHub workflow, or run the CLI locally with npx maintainer-signal --repo owner/name.

Review the checked-in sample report to see the exact Markdown output before installing it in your own repository.
